Guidance

Purpose

This tool is designed to:

  • Provide up-to-date information on several commonly used indices, from 2008 onwards. It also includes short-term future projections.

  • Enable user-generated indices to be used throughout the rest of the application.

    • Background

  • Indices show how prices have, in general, increased (or decreased) over time.

  • A price index of 100 represents the 'base period', to which all other periods are referenced to. For example, a value of 105 represents that this period's prices are 5% higher than in the base period.

    • Indices: User Guidance

  • User Parameters:

    • - Index: select an index from the dropdown menu.

    - Period Reference: select the period reference of interest (financial year, calendar year, or quarterly).

    - Base Period: select the relevant base year. Default varies by index. Note this cannot be set before 2008.

  • Interpretation:

    • - Period: based upon the chosen period reference.

    - Index: displays the chosen indices' values for every available year, dependent on the base period.

    - YoY (%): displays the year-on-year percentage change from the current period to the previous period. Note that quarterly periods refer to the same quarter from a year prior.

  • Notes:

    • - Highlighted Rows:

      a. Yellow: periods for which indices are forecasts.

      b. Blue: period which is the base year (if applicable).

      c. Green: period which is both the base year and a forecast (if applicable).

  • Download Options:

    • - Download displayed data: downloads only the displayed data, tailored to user-selected parameters.

    - Download raw data: downloads the complete raw data set, with no manipulation. This includes all indices, all period references, and original base years.

    Add Index: User Guidance

  • User Parameters:

    • - Index Name: provide the index name.

    - Period Reference: specify the period reference (e.g. Financial Year, Calendar Year, Quarterly).

    - Start Period: define the start period for the user-generated index.

    - End Period: define the end preiod for the user-generated index.

    - Generate Table: updates the table based on user parameters.

    - Update Local Index Set: adds the user-generated index to local set.

  • Notes:

    • a. Index Name: must be supplied, and must be named different to other user-generated indices.

    b. User-generated indices will not be saved upon closing the application.

    c. Table Options: a complete set of Index values, or year-on-year percentage changes must be supplied (or both), and the following conditions apply:

    i. Index: if supplied only, the year-on-year changes will be automatically generated, with the start period year-on-year change being equal to 0.

    ii. Year-on-year: if supplied only, the index values will be automatically generated, with the start period index being equal to 100 (unless otherwise specified).
