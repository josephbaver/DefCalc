Guidance

Purpose

This tool is designed to:

  • Inflate (and deflate) financial values to generate an estimate of required future (or current equivalent) expenditure, utilising a range of user-input controls.

  • Convert between nominal and real financial values, and to rebase real values from one time period to another.

  • Estimate the real or nominal percentage changes for each period relative to a base-year (or the previous period).

    • Background

  • Inflation is used to convert from real values and nominal values, and accounts for the effect of inflation. Negative inflation is known as deflation.

  • Deflating converts nominal values to real values (i.e. removes the effect of inflation), in line with HMT guidance.

  • It is important to know whether values are nominal or real before using this tool to ensure the appropriate inputs can used.

  • In public cost-benefit analyses, it is usually recommended to use HMT's GDP deflator for converting nominal values to real values. This tool provides this as the default option.

  • Note that any forecasts going beyond the years specified in the 'Indices Tool' are forecast using a flat rate of inflation equal to the last year-on-year change available.

    • User Guidance

  • User Parameters:

    • - Input Table: Input, either manually or copy-paste* from a compatable spreadsheet (Ctrl + C, CTRL + V), financial values into the input table. Only cells with values will be converted (e.g. to inflate £100 from 2010 onwards, £100 must be entered from 2010 onwards).
    *This function is not supported by Internet Explorer.

    - Index: select an index from the dropdown menu.

    - Conversion (Real/Nominal): select whether to convert from real-to-nominal (inflate) or nominal-to-real (deflate).

    - Convert From (Base Period): select the relevant period to use as the base for conversion. Note this cannot be set before 2008.

    - Period Reference: select the period reference of interest (financial year, calendar year, or quarterly).

    - Start Period: provide the relevant first period to display. This will affect the outputs, so select carefully. Note this cannot be set before 2008.

    - End Period: provide the relevant final period to display. This will affect the outputs, so select carefully.

    - Number of Required Rows: Number input. Note that arbitrarily large numbers may cause lagged responses or crash the application.

    - Percentage Change: select chosen percentage change comparison; either base-to-period (i.e. all relative to base period), or period-to-period (i.e. relative to prior period).

    - Update Table: when parameters are set up, this button will generate an updated table.

    - Download Results: download output results, tailored to user-selected parameters.

  • Interpretation:

    • - Output Table: displays the converted results of the input table, based on the user parameters.

    - Percentage Change: displays the converted results as percentage changes, referencing either the base-to-period or period-to-period parameter.

  • Reporting Results:

    • - Be explicit in reporting which index is used.

    - Real/Nominal conversions: be explicit in the years that are converted to/from.

    - Re-basing: be explicit in which year(s) that are converting from, and which year is the new base year.

    - For example: [these values] are nominal prices. They were inflated, from 20XX prices; or [these other values] are real prices. They were deflated, and are measured in 20XX prices.

  • Download Options:

    • - Download Results: displays the input table, followed by the output table, and then the conversion factor used.
